edit and render *.md files

On all desktop hosts we have retext installed. retext is a powerful editor which provides a live preview (press: Ctrl+E) and a side by side live preview ([ress: Ctrl+L).

With mc you will get a preview if you press F3 (system wide changed config).

You can manually render the preview on a terminal using the following command:

pandoc -s -f markdown -t man README.md | groff -T utf8 -man | less

Very usefull is an alias mdless which you can but in your ~/.bashrc:

alias mdless "pandoc -s -f markdown -t man \!* | groff -T utf8 -man | 
less"

You can configure Kate to run pandoc to compile a html page and open it with a webbrowser:

  • in the Menu got to 'Settings' -> 'Configure Kate' -> 'Plugins'
    and activate 'Build Plugin'
  • click 'Build Output' at the bottom Menu and go to 'Target Settings'
    • at the following command to 'Quick compile':
      'pandoc -r markdown -w html -o %f.html %f; konqueror %f.html'
  • now you can choose 'Quick Compile' in the Menu 'Build' and you get a new Konqueror window with the compiled Markdown.
  • configure a shortcut for 'Quick Compile'

use the webinterface from home

Because of admin repositories which need anonymous read only access we block the webinterface at our firewall. There is the possibility to open a socks proxy over ssh to overcome this.

Steps necessary:

  • open the socks proxy over a ssh tunnel to sim1 (no prompt, Ctrl+c to stop):
ssh ${username}@${host_name_spsc}.spsc.tugraz.at -ND1080
  • configure the webbrowser to use the socks on localhost:1080

or start the webbrowser google-chrome with a parameter to overwrite the default settings this will only work if no instance is already running!

google-chrome --proxy-server="socks5://127.0.0.1:1080" 
--host-resolverules="MAP * 0.0.0.0 , EXCLUDE 127.0.0.1" 
https://gitint.spsc.tugraz.at

Use the Web-Interface from Home (Mac + IceCat)

Make Shell-Script

The following procedures must be done only once:

  • download GNU IceCat (at least as good as Safari)

  • set 'socks host' in 'preferences' -> 'advanced' -> 'network' -> 'settings' -> 'manual proxy configuration' according to

    • socks host: localhost
    • port: 8080

  • make a shell-script that contains the following code (copy&paste): 

#!/bin/bash

# Type in your username. 
user_name_spsc=<USERNAME> 
host_name_spsc="sim1" 

# Open ssh-tunnel. 
echo "User: ${user_name_spsc}"
echo "Host: ${host_name_spsc}"
ssh ${user_name_spsc}@${host_name_spsc}.spsc.tugraz.at \
-D 8080 -f -C -q -N

# Open IceCat. 
echo "Make sure that IceCat's 'socks proxy server' \
      is set to 'localhost:8080' if websites don't load." 
sleep 3s
open -a IceCat \
     http://ganglia.spsc.tugraz.at \
     https://gitint.spsc.tugraz.at \
     https://www.spsc.tugraz.at \
     https://calendar.google.com/calendar/htmlembed?src=ilona.poelzl@tugraz.at

This script opens a proper ssh-tunnel and the following, most important websites: 'gitint.spsc.tugraz.at', 'ganglia.spsc.tugraz.at', 'www.spsc.tugraz.at', and the internal SPSC-calender.

Call Shell-Script in Terminal

Just call the shell-script in your terminal, enter your SPSC-password, and IceCat opens automatically with the websites (as tabs) mentioned above.

Use the Web-Interface from Home (Mac + Safari)

The following procedures must be done every time (due to the manual change of network-properties):

  • Close all open instances of your web-browser.
  • Open advanced-section of your OS’s network-properties.
    • Check box 'socks proxy'.
    • Set 'socks proxy server' to 'localhost:8080'.
  • Open Terminal.
    • Type ssh {USERNAME}@sim1.spsc.tugraz.at -D 8080 -f -C -q -N and enter password.
    • There shouldn’t be any prompt after entering the password.
    • Type open -a {BROWSER} https://gitint.spsc.tugraz.at to open GitLab.
    • Type open -a {BROWSER} http://ganglia.spsc.tugraz.at to open Ganglia (no https).
    • If command OPEN cannot open web-browser, use full path, e.g., /Applications/{BROWSER}.app.

Example (Safari):

  • Type open -a safari https://gitint.spsc.tugraz.at to open GitLab.
  • Type open -a safari http://ganglia.spsc.tugraz.at to open Ganglia.

If you have any trouble with accessing GitLab or Ganglia from your Mac, feel free to contact Hannes.
